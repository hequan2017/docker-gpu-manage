### 后端文件修改

#### 1. 菜单配置 (`server/source/system/menu.go`)

```go
{MenuLevel: 0, Hidden: false, ParentId: 0, Path: "project", Name: "project",
```

```go
menu.Name == "project"  // 新增项目说明菜单
```

## 🚀 部署步骤

### 方式一：已有系统升级

如果您的系统已经在运行，需要手动更新数据库：

#### 1. 登录系统
使用管理员账号登录系统（admin / 123456）

#### 2. 添加菜单
进入 **超级管理员 > 菜单管理**，点击"新增按钮"，添加以下菜单：

```json
{
  "path": "project",
  "name": "project",
  "parentId": 0,
  "title": "项目说明",
  "icon": "document",
  "component": "view/about/project.vue",
  "sort": 2,
  "menuLevel": 0,
  "hidden": false
}
```

#### 3. 配置权限
进入 **超级管理员 > 角色管理**，为需要的角色（如普通用户8881）添加"项目说明"菜单权限。

#### 4. 部署前端文件
将新创建的 `project.vue` 文件部署到前端服务器的对应目录：
```bash
# 复制文件到生产环境
cp web/src/view/about/project.vue /path/to/production/web/src/view/about/

# 重新构建前端
cd /path/to/production/web
npm run build
```

### 方式二：全新系统部署

如果是全新系统，重新初始化数据库即可自动生成菜单和权限：

#### 1. 启动后端服务
```bash
cd server
go run main.go
```

#### 2. 访问前端进行数据库初始化
访问 `http://localhost:8080`，按照页面提示完成数据库初始化。

系统会自动：
- 创建所有菜单（包括"项目说明"菜单）
- 配置所有角色的菜单权限
- 创建默认管理员账号

#### 3. 启动前端服务
```bash
cd web
npm install
npm run dev
```

### 方式三：Docker 部署

如果使用 Docker 部署，需要重新构建镜像：

```bash
cd deploy/docker-compose

# 重新构建并启动
docker-compose down
docker-compose build
docker-compose up -d
```

## 📝 注意事项

1. **图片资源**
   - 确保 `docs/` 目录下有项目截图
   - 图片文件名：1.png, 2.png, 3.png, 4.png, 5.png
   - 如果图片不存在，会显示占位图标

2. **图标依赖**
   - 页面使用了 Element Plus Icons
   - 确保项目中已安装 `@element-plus/icons-vue`

3. **浏览器兼容性**
   - 推荐使用现代浏览器（Chrome、Firefox、Edge）
   - 需要支持 ES6+ 语法

4. **性能优化**
   - 图片建议使用压缩后的版本
   - 大图片可能影响页面加载速度

## 🐛 常见问题

### Q1: 点击菜单后页面404
**A**: 检查以下几点：
1. 确认 `project.vue` 文件是否在正确位置
2. 检查前端是否重新构建
3. 清除浏览器缓存重试

### Q2: 图片无法显示
**A**:
1. 确认图片文件是否存在于 `public/docs/` 目录
2. 检查图片文件名是否正确
3. 查看浏览器控制台的错误信息

### Q3: 菜单不显示
**A**:
1. 确认当前用户角色是否有该菜单权限
2. 检查菜单的 `hidden` 字段是否为 `false`
3. 重新登录刷新权限

### Q4: 样式显示异常
**A**:
1. 确认 Element Plus 样式是否正常加载
2. 检查是否有样式冲突
3. 清除浏览器缓存
